# BlueDot Intelligence UI 规范

本规范基于当前演示版页面、PRD 背景与现有 `src/styles.css` 的紧凑密度实现整理，用于后续首页、资源中心、实例中心、待发布、端口白名单、告警中心与审计日志保持统一。当前产品定位是技术运维视角的 AI 算力资源管理端，界面应偏高密度、可扫描、可操作，避免营销化大标题和装饰化排版。

## 1. 设计定位

- 产品类型：AI 算力 / GPU vGPU 资源管理后台。
- 用户角色：平台管理员、技术运维、资源调度人员。
- 视觉关键词：冷静、清晰、高密度、轻量科技感、低装饰。
- 信息优先级：资源状态 > 风险告警 > 实例调度 > 审计留痕。
- 交互原则：前台不做手动 GPU 拆分，不单独分配具体物理 GPU；所有 GPU 统一进入 vGPU 后台调度资源池。

## 2. 布局规范

### 2.1 页面骨架

- 左侧固定侧边栏：宽度 `260px`。
- 顶部栏：高度约 `82px`，仅保留标题、搜索、语言切换、添加虚拟数据、创建实例、用户菜单；草稿箱入口暂隐藏，待发布保留在实例中心二级页面。
- 主内容区：桌面端左右内边距 `30px`，顶部内边距 `28px`，底部内边距 `46px`。
- 内容最大宽度：沿用当前 `1720px`，居中展示，适配宽屏。
- 页面纵向模块间距：`22px`。

### 2.2 栅格

- 首页指标卡：4 等分栅格，卡片间距 `20px`。
- 首页主内容：左侧趋势图 + 右侧状态/告警，间距 `20px`。
- 资源中心：显卡资源与资源概览采用 2 列，数据表盘内部最多 5 列。
- 实例中心：列表使用横向自适应表格，桌面端保证关键列不换行；中小屏允许降为两列/单列堆叠。
- 弹窗：创建实例弹窗最大宽度约 `1200px`，内部左侧基础信息、右侧推荐与资源预检上下排列；移动端改为单列。

## 3. 字体规范

### 3.1 字体族

统一使用系统无衬线字体：

```css
Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif
```

### 3.2 字号层级

| 场景 | 字号 | 字重 | 说明 |
| --- | ---: | ---: | --- |
| 顶部产品标题 | `25px` | `800-900` | `AI 算力资源管理端` |
| 页面标题 | `24px` | `800-900` | 资源中心、推理实例部署与运维、待发布 |
| 卡片标题 | `20px` | `800-900` | 图表、列表、模块标题 |
| 列表主文本 | `17px` | `800-900` | 实例名称、重要资源规格 |
| 正文 / 描述 | `14px` | `400-600` | 卡片说明、描述文本 |
| 辅助文本 | `13px` | `600-800` | 标签、元信息、表格内容 |
| 徽标 / 小按钮 | `12-13px` | `800-900` | 状态、数量、操作标签 |
| 大数字指标 | `30-36px` | `900` | 首页、资源概览、健康分 |

### 3.3 行高与文本

- 正文行高：`1.55`。
- 长说明文本不使用大段落，优先 1 到 2 行。
- 表格、列表、卡片内不使用英雄级字号。
- 数据列建议使用紧凑文本和等宽/半等宽数字视觉，避免列宽抖动。

## 4. 间距规范

采用 `4px` 基础步进，常用间距如下：

| Token | 数值 | 用途 |
| --- | ---: | --- |
| `space-1` | `4px` | 图标和短文本微间距 |
| `space-2` | `8px` | 标签组、二级菜单间距 |
| `space-3` | `12px` | 小卡片内部元素间距 |
| `space-4` | `16px` | 表单、列表项内部标准间距 |
| `space-5` | `20px` | 卡片间距、详情区间距 |
| `space-6` | `24px` | 卡片内边距、模块头部 |
| `space-7` | `28px` | 页面内容顶部留白 |
| `space-8` | `30px` | 主内容左右留白 |

关键落点：

- 卡片头部：`22px 24px 16px`。
- 普通卡片内容：`24px`。
- 指标卡：`22px 24px`。
- 表格单元格：`14px 22px`。
- 实例行：`18px 20px`。
- 列表操作按钮间距：`7-8px`。
- 二级菜单：左缩进约 `38px`，行距 `8px`。

## 5. 色彩规范

### 5.1 核心色

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 主强调色 | `#b8f155` | 创建实例、进度、关键 CTA |
| 主文本 | `#18181b` | 标题、重要数据 |
| 深色按钮 / 激活态 | `#171717` / `#191919` | 侧边栏选中、分段控件选中 |
| 页面背景 | `#f7f8f7` | 主工作区背景 |
| 卡片背景 | `#ffffff` | 主要承载面 |
| 弱背景 | `#fbfbfb` / `#fbfcf8` | 信息块、健康卡 |
| 边框 | `#e1e5dc` / `#e2e5eb` | 卡片、输入框、表格 |
| 次级文本 | `#788293` / `#8a93a3` | 描述、辅助信息 |
| 成功 | `#16a34a` | 运行中、通过 |
| 警告 | `#d97706` / `#f97316` | 排队、资源检测中 |
| 危险 | `#ef4444` / `#dc2626` | 失败、释放、告警 |
| 信息 | `#2563eb` | 扩容观察、排队提示 |

### 5.2 使用规则

- 页面不能被单一绿色铺满，荧光绿只用于主动作、进度、少量状态强调。
- 黑色用于强选中态和主操作对比，避免大面积黑底。
- 红色只用于危险操作、失败、严重告警；删除/释放必须配确认弹窗。
- 状态色必须搭配文字，不只靠颜色传达含义。

## 6. 圆角与阴影

- 普通卡片圆角：`14px`。
- 指标卡圆角：`14px`。
- 小信息块圆角：`8-12px`。
- 按钮圆角：`8px`，紧凑操作按钮 `9px`。
- 标签圆角：胶囊形或 `7-8px`。
- 创建弹窗圆角：`20px`。
- 确认弹窗圆角：`18px`。
- 阴影应轻：卡片使用低透明度大阴影，弹窗使用更强遮罩与投影；不要增加装饰性发光背景。

## 7. 组件规范

### 7.1 顶部栏

- 标题固定为 `AI 算力资源管理端`。
- 不展示 `系统在线 · LAST UPDATE ...`。
- 搜索框文案：`搜索实例、节点、端口、告警`。
- 顶部暂不展示草稿箱按钮，待发布入口位于实例中心二级导航。
- 主 CTA 为荧光绿 `创建实例`。
- 用户菜单显示 `admin`。

### 7.2 侧边栏

- 一级导航：首页、资源中心、实例中心、端口白名单、告警中心、审计日志。
- 实例中心二级：推理与部署、待发布。
- 删除一级草稿箱入口。
- 当前页使用黑色选中背景，右侧可保留荧光绿状态点。
- 底部健康卡固定展示 GPU 虚拟化健康分和风险摘要。

### 7.3 卡片

- 卡片用于承载单一信息组，不做卡片套卡片。
- 卡片头部必须包含清晰标题，可带一行辅助说明。
- 信息卡应可快速扫描，优先使用数字、状态、短标签。
- 首页和资源中心保持高密度，避免过大的空白和营销式布局。

### 7.4 表格与列表

- 表头字号 `13px`，颜色偏浅，必要时大写或字距，但不要过度拉开。
- 行高紧凑，实例行标准 padding `18px 20px`。
- 操作按钮放在行尾，顺序建议：日志、停止、重启、释放。
- 破坏性操作使用红色描边或红色实心确认按钮。
- 长文本使用省略号，详情通过展开区域查看。

### 7.5 状态徽标

- 运行中：绿色。
- 排队中 / 等待中 / 部署中 / 资源检测中：黄色。
- 失败 / 异常中：红色。
- 已停止 / 草稿 / 待发布：灰色。
- 扩容观察中：蓝色。
- 徽标高度保持紧凑，文字 `12-13px`。

### 7.6 表单

- 输入框高度 `48px`。
- 表单标签必须显式展示，不只依赖 placeholder。
- 创建实例保留基础信息、部署类型、资源设备、设备数量、创建人、请求 ID、请求序列、请求上下文、K8s 资源名与 NodePort 策略。
- 镜像地址、容器端口、NodePort、Deployment 名由后端默认模板或自动生成，前端不做手动配置。
- 推荐方案只展示系统推荐，不提供手动拆分策略。
- 资源预检结果包括资源设备、CPU / 内存余量、NodePort 自动避让、部署锁与并发校验、创建人审计标记、vGPU 预测占用。

### 7.7 弹窗与抽屉

- 创建实例使用大弹窗，最大宽度约 `1200px`。
- 停止、重启、释放使用小型确认弹窗，宽度约 `560px`。
- 日志使用右侧抽屉，宽度约 `720px`。
- 遮罩透明度保持中等，不能让背景完全消失。
- 弹窗按钮文案：取消使用 `Cancel` 或 `取消` 需全项目统一；当前参考图保留 `Cancel`。

## 8. 页面级规范

### 8.1 首页

必须包含：

- 集群健康、运行卡数、显存利用率、封闭/白名单端口等核心指标。
- 资源趋势图。
- 实例状态分布。
- 部署实例管理摘要。
- 实时告警摘要。

首页不是营销页，不使用大 Hero。

### 8.2 资源中心

必须体现 vGPU 后台调度方向：

- 显卡资源：vGPU 分配率、算力分配率、显存分配率、算力使用率、显存使用率。
- 资源概览：节点、显卡、vGPU、算力、显存大小。
- 显卡类别占比。
- 节点资源分配率 Top5。
- vGPU 后台调度拓扑。
- 显卡信息列表。

显卡只作为后台资源池基础单元展示，实例不直接绑定具体物理 GPU。

### 8.3 实例中心 / 推理与部署

- 顶部样式与待发布、资源中心一致。
- 不展示 `BlueDot Intelligence / 实例中心` 面包屑。
- 列表必须自适应，并支持展开只读详情。
- 操作包含日志、停止、重启、释放。
- 运维操作必须进入确认弹窗或抽屉，并写入审计语义。

### 8.4 待发布

- 承接原草稿箱内容。
- 作为实例中心二级页面，不作为一级导航。
- 保存到待发布不占用 GPU/NPU 资源。
- 操作包括继续编辑、从待发布创建、删除草稿。

### 8.5 告警中心

- 保持指标卡 + 异常类别 + 层级筛选 + 实时告警 + 历史记录结构。
- 告警类别必须包含 vGPU 资源超限、峰值重叠风险、配额不足等虚拟化异常。
- 实时告警项必须包含推荐处理动作。

### 8.6 审计日志

- 保留虚拟卡推荐、策略创建、告警处理、实例操作、端口操作等留痕。
- 表格优先可扫描，操作对象用等宽或弱强调显示。

## 9. 交互规范

- 主要按钮最小高度 `44px`，紧凑行内按钮可 `36px`，但点击间距不少于 `8px`。
- 分段控件选中态使用黑底白字。
- Hover 只做轻微背景变化，不依赖 hover 才能发现功能。
- 破坏性操作必须二次确认。
- 日志抽屉支持搜索、复制、自动滚动。
- 资源不足时优先进入排队；高优先级任务排队时优先处理；共享不可用或配置未完成时保存待发布。

## 10. 响应式规范

- `1280px` 以下：主内容栅格转单列，实例表格转两列信息流。
- `760px` 以下：侧边栏改为普通流式区域，所有关键栅格转单列。
- 移动端禁止出现页面级横向滚动；表格确实过宽时仅表格容器允许横向滚动。
- 小屏下弹窗按钮可纵向排列，日志工具栏改为单列。

## 11. 可访问性与可维护性

- 图标按钮需要明确 `aria-label` 或可见文字。
- 状态不可只依赖颜色，必须有文字。
- 交互控件保持可键盘聚焦。
- 禁止新增无意义装饰图形、渐变球、过度模糊背景。
- 新增样式优先抽象为复用类或 token，不要在组件里散落 raw hex。
- 后续建议把当前颜色、字号、间距沉淀为 CSS 变量，减少重复覆盖。

## 12. 走查清单

提交 UI 改动前按以下清单检查：

- 顶部栏是否仍为紧凑高度，且没有恢复系统在线时间字段。
- 页面标题是否为 `24px` 左右，未出现过大 Hero 字号。
- 卡片间距是否保持 `20-22px`，卡片内边距是否约 `24px`。
- 表格行是否紧凑，操作按钮是否没有撑高行高。
- 实例中心二级菜单行间距是否清晰但不松散。
- 资源中心是否表达 vGPU 后台统一调度，而不是手动拆分 GPU。
- 危险操作是否使用确认弹窗。
- 徽标、进度、图表颜色是否语义一致。
- 低于 `1280px` 和 `760px` 时是否无内容重叠。
- 新增页面是否遵循已有侧边栏、顶部栏、卡片、按钮规范。
